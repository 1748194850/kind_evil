## 正确的文件结构

### 项目根目录应该包含：

```
kind_evil/
├── .editorconfig              # ✅ 代码风格配置
├── CHANGELOG.md              # ✅ 变更日志
├── .github/                   # ✅ GitHub 配置
│   ├── workflows/
│   │   └── unity-build.yml
│   └── ISSUE_TEMPLATE/
│       ├── bug_report.md
│       └── feature_request.md
├── Assets/                    # Unity 资源目录
│   ├── Scenes/
│   ├── 中文文档/
│   └── ...
├── Library/                   # Unity 自动生成（不应提交）
├── Packages/                  # Unity 包管理
├── ProjectSettings/           # Unity 项目设置
└── .gitignore                # Git 忽略文件
```

### Assets 目录应该包含：

```
Assets/
├── Scenes/                    # 场景和脚本
├── Materials/                 # 材质
├── Prefabs/                   # 预制体
├── Textures/                  # 纹理
├── Scripts/                   # 独立脚本（如需要）
└── 中文文档/                  # 项目文档
```

## 为什么这些文件不应该在 Assets 目录？

1. **`.editorconfig`**
   - 这是项目级别的配置文件
   - IDE 和编辑器会在项目根目录查找此文件
   - Unity 会为 Assets 下的文件生成 `.meta` 文件，这是不必要的

2. **`CHANGELOG.md`**
   - 这是项目文档，不是 Unity 资源
   - 应该在项目根目录，便于版本控制工具识别
   - Unity 会为它生成 `.meta` 文件，这是不必要的

3. **`.github/`**
   - GitHub Actions 和 Issue 模板需要在项目根目录
   - GitHub 会在项目根目录查找这些配置
   - Unity 会为目录下的文件生成 `.meta` 文件，这是不必要的

## 注意事项

1. **不要将配置文件放在 Assets 目录**
   - 配置文件（`.editorconfig`、`CHANGELOG.md`、`.github/` 等）应该在项目根目录
   - Unity 会为 Assets 下的所有文件生成 `.meta` 文件，这是不必要的

2. **Unity 资源应该在 Assets 目录**
   - 场景文件（`.unity`）
   - 脚本文件（`.cs`）
   - 材质、纹理、预制体等资源

3. **项目配置文件应该在项目根目录**
   - `.editorconfig`
   - `CHANGELOG.md`
   - `.github/`
   - `.gitignore`
   - `README.md`（如果不在 Assets 下）

